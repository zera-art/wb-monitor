# 📊 WB Monitor — Система управления оборачиваемостью Wildberries

Автоматическая система мониторинга остатков, оборачиваемости и принятия решений по SKU.

---

## Архитектура

```
WB API (6 блоков)
    ↓  (каждый день в 07:00 UTC)
Python ETL (main.py)
    ↓
Google Sheets Dashboard     +     Telegram-уведомления (опц.)
  🎯 РЕШЕНИЯ                         • Срочные SKU
  📊 СВОДКА                          • Дайджест
  📦 ВСЕ SKU
  📅 ИСТОРИЯ
  ⚙️ НАСТРОЙКИ
```

## Быстрый старт

### Шаг 1 — Клонировать репозиторий

```bash
git clone https://github.com/YOUR_USERNAME/wb-monitor.git
cd wb-monitor
pip install -r requirements.txt
```

### Шаг 2 — Настроить Google Sheets

1. Перейдите в [Google Cloud Console](https://console.cloud.google.com)
2. Создайте проект → **APIs & Services → Enable APIs**
3. Включите:
   - **Google Sheets API**
   - **Google Drive API**
4. Создайте **Service Account**:
   - IAM & Admin → Service Accounts → Create
   - Роль: Editor
   - Создайте JSON-ключ → скачайте как `google_credentials.json`
5. Создайте новую Google Таблицу
6. Поделитесь таблицей с email сервис-аккаунта (из JSON-файла, поле `client_email`)
   - Права: **Редактор**
7. Скопируйте ID таблицы из URL:
   `https://docs.google.com/spreadsheets/d/**SPREADSHEET_ID**/edit`

### Шаг 3 — Настроить API ключи WB

В личном кабинете WB (Настройки → Доступ к API) создайте ключи:

| Ключ          | Необходимые права                                    |
|---------------|------------------------------------------------------|
| `stats`       | Статистика: остатки, продажи, отчёты, хранение       |
| `content`     | Контент: чтение карточек                             |
| `prices`      | Цены: чтение и изменение цен                         |
| `analytics`   | Аналитика: аналитика карточек                        |
| `ads`         | Реклама: чтение статистики кампаний                  |

### Шаг 4 — Конфигурация

#### Вариант A: .env файл (для локального запуска)

```bash
cp .env.example .env
# Заполните .env своими ключами
```

#### Вариант B: Прямо в config.py

```python
WB_KEYS = {
    "stats":     "ваш_ключ_статистики",
    "content":   "ваш_ключ_контента",
    "prices":    "ваш_ключ_цен",
    "analytics": "ваш_ключ_аналитики",
    "ads":       "ваш_ключ_рекламы",
}
SPREADSHEET_ID = "ваш_id_таблицы"
```

### Шаг 5 — Первый запуск

```bash
# Проверка без записи в Sheets
python wb_monitor/main.py --dry-run

# Полный запуск
python wb_monitor/main.py
```

## Автоматизация через GitHub Actions

1. Загрузите код в GitHub
2. Добавьте секреты (Settings → Secrets → Actions):

| Секрет                   | Значение                                  |
|--------------------------|-------------------------------------------|
| `WB_KEY_STATS`           | Ключ статистики WB                        |
| `WB_KEY_CONTENT`         | Ключ контента WB                          |
| `WB_KEY_PRICES`          | Ключ цен WB                               |
| `WB_KEY_ANALYTICS`       | Ключ аналитики WB                         |
| `WB_KEY_ADS`             | Ключ рекламы WB                           |
| `SPREADSHEET_ID`         | ID Google Таблицы                         |
| `GOOGLE_CREDENTIALS_JSON`| Содержимое google_credentials.json        |
| `TELEGRAM_ENABLED`       | `true` или `false`                        |
| `TELEGRAM_BOT_TOKEN`     | Токен бота (опц.)                         |
| `TELEGRAM_CHAT_ID`       | ID чата (опц.)                            |

3. Workflow запускается автоматически каждый день в 10:00 МСК
4. Можно запустить вручную: Actions → WB Monitor → Run workflow

## Структура дашборда

### 🎯 РЕШЕНИЯ
Главный рабочий лист. Показывает только то, что требует действий:
- 🚨 **СРОЧНЫЕ** — критичные остатки, мёртвые товары
- ⚠️ **ВАЖНЫЕ** — замедленная оборачиваемость, высокий ДРР
- 🔍 **МОНИТОРИНГ** — нормализующиеся товары, готовые к повышению цены

### 📊 СВОДКА
- KPI за неделю: выручка, хранение, реклама, ДРР
- Разбивка по статусам
- Топ-5 по стоимости хранения

### 📦 ВСЕ SKU
Полная таблица всех позиций с метриками и рекомендациями.

### 📅 ИСТОРИЯ
Недельные снимки — для анализа трендов и динамики оборачиваемости.

### ⚙️ НАСТРОЙКИ
Пороговые значения системы классификации.

## Статусы SKU

| Статус | Условие | Рекомендация |
|--------|---------|--------------|
| 🔴 КРИТИЧНЫЙ ОСТАТОК | Оборачиваемость > 90 дней | Срочно снизить цену, усилить рекламу |
| 🔴 МЁРТВЫЙ ОСТАТОК | Продаж нет 4+ недели | Дропнуть цену -50% или самовыкуп |
| ⚠️ РАСПРОДАЖА НЕЭФФЕКТИВНА | Скидка >25% + нет роста продаж | Проблема в карточке, не в цене |
| 🟠 ЗАМЕДЛЕННАЯ ОБОРАЧИВАЕМОСТЬ | 60–90 дней | Снизить цену 10-15% или запустить рекламу |
| 💸 ВЫСОКИЙ ДРР | ДРР > 25% | Снизить ставки, проверить конверсию |
| ✅ НОРМАЛИЗАЦИЯ | Продажи растут + оборачив <30д | Готовиться к повышению цены |
| 🟢 ПОДНЯТЬ ЦЕНУ | Оборачиваемость < 14 дней | Тестировать +5% каждые 3 дня |
| 📈 МАСШТАБИРОВАТЬ РЕКЛАМУ | ДРР < 10% | Увеличить бюджет на 20-30% |
| 🟡 МОНИТОРИНГ | Норма | Продолжать наблюдение |

## Telegram-уведомления (опционально)

1. Создайте бота через @BotFather → получите TOKEN
2. Узнайте свой CHAT_ID через @userinfobot
3. Установите `TELEGRAM_ENABLED=true` в GitHub Secrets

Каждый день в 10:00 МСК вы получаете дайджест:
- Выручка, хранение, ДРР за неделю
- Список срочных SKU с рекомендациями

## Настройка порогов

Порogi классификации можно изменить в `config.py`:

```python
THRESHOLDS = {
    "critical_turnover_days": 90,    # Повысьте до 60 для более строгого контроля
    "slow_turnover_days": 60,
    "price_raise_turnover_days": 14, # Товары быстро улетают — когда поднимать цену
    "ad_drr_too_high": 0.25,         # 25% ДРР — предел рентабельности
    "ad_drr_good": 0.10,             # 10% ДРР — масштабировать рекламу
}
```
